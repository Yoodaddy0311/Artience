# PRD 부록 — Studio UX 가이드(초안)

## Studio 모드 기본 UX 원칙
- 사용자는 “그리기”가 아니라 “선택/수정요청/적용”을 한다.
- Draft는 항상 안전하다(프로젝트에 영향 없음).
- Apply는 원자적(한 번에 반영)이고, 항상 되돌릴 수 있다(스냅샷).

## 패널 구성(권장)
- Inbox: 파일 목록 + 태그 + 미리보기
- Builder: 채팅 + “생성 범위 선택”(Theme/World/Agents/Recipes)
- Draft: 이번 생성 결과의 요약 + Preview 이동
- Versions: 최신 버전 목록(적용자/시간/요약) + Rollback + Compare

## AI Builder 입력 가이드(사용자에게 보여줄 문구 예시)
- “회의실 2개, 업무존 1개, 휴게실 1개. 미니멀 톤. 동선 막히지 않게.”
- “에이전트 25명. 역할은 PM/Dev/QA/Writer/Analyst로 분배.”
- “레시피 3개: 코드 생성, 테스트, 문서화.”

## 버전 요약 생성(자동)
- Apply 시 변경 요약 자동 생성:
  - “회의실 추가(1→2)”
  - “충돌 타일 감소(120→80)”
  - “레시피 추가: 문서화”
