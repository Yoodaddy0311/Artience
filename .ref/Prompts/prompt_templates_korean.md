# AI Builder 프롬프트 템플릿(Studio 모드용)

권장 형식: (1) 목표 1문장 → (2) 제약 3~5개 → (3) 산출물 형식 지정 → (4) 수정 요청 규칙

## 1) 오피스 레이아웃 생성
“모던/미니멀 오피스 레이아웃을 생성해줘. 방은 회의실 2, 업무존 1, 휴게실 1, 리셉션 1. 동선이 막히지 않게 문과 복도를 확보하고, 스폰 포인트는 3개 이상. 결과는 project.json의 world.layers로 적용 가능한 Draft로.”

## 2) 레이아웃 수정(동선)
“현재 레이아웃에서 스폰(5,5)에서 이동이 막혀. collision 타일을 최소화해서 모든 방으로 이동 가능하게 수정해줘. 벽은 유지하고 문 위치만 재배치해도 돼.”

## 3) 테마 생성(팔레트)
“다크 모드 기반이지만 답답하지 않은 테마를 만들어줘. bg/panel/text/accent 팔레트를 제안하고, accent는 ‘성공’에 어울리게. 산출물은 theme Draft로.”

## 4) 에이전트 구성(25명)
“에이전트 25명을 생성해줘. 역할은 PM 3, Dev 12, QA 5, Writer 3, Analyst 2. 홈 좌표를 업무존 주변에 배치하고, 이름은 Agent-01~25.”

## 5) 레시피 생성
“레시피 3개를 만들어줘: (1) 코드 스캐폴딩 (2) 테스트 실행 (3) 문서 요약. command/args/cwd/env 포함. 로그 키워드로 THINKING/RUNNING 전환 규칙도.”

## 6) 로그 요약
“Logs 탭에 raw 로그 외에 ‘한국어 요약(5줄)’ 영역을 추가. 에러면 원인 후보 3개 bullet.”

## 7) 가구 배치
“업무존에 책상 12, 의자 12. 회의실에 테이블 1, 의자 6. 충돌은 책상/테이블만, 의자는 통과.”

## 8) 말풍선 상호작용(스텁)
“회의실에 2명 이상 들어오면 말풍선 ‘sync’ 2초 표시. 대화는 MVP 스텁.”

## 9) seed 기반 생성
“Generate Office에 seed 입력 추가, 같은 seed면 같은 결과. seed는 문자열도 허용.”

## 10) 에셋팩 적용
“ZIP 에셋팩(pack.json)을 읽어 타일/오브젝트/스프라이트 등록. Assets 탭에서 선택 적용.”
